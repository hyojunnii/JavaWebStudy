Get방식으로 요청 후 응답페이지 확인

특징1. GET방식으로 요청시 URL의 Header영역에 데이터들을 포함시켜서 요청
- 사용자가 입력한 값(데이터)들이 url에 노출됨
- 보안유지가 불가능
- 로그인이나 회원가입같은 경우 GET방식에 부적합함

특징2. Header영역은 전송하는 길이에 제한이 있음
- 방대한 데이터 담았을 경우, 초과된 데이터는 절단되어 넘어감
- 게시판 작성 같은 경우 GET방식 부적합

특징3. 장점이라고 한다면 url에 데이터가 노출되어있기 때문에 즐겨찾기(북마크)(url저장해두었다가 요청)에 적합
- 검색 기능 같은 GET방식에 적합
